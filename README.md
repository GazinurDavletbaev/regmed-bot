# RegMed Bot – Telegram-бот с RAG для лаборатории медицинских изделий

## Архитектура
Чистая архитектура (Clean Architecture) с разделением на слои:

### Слои
- **`presentation/telegram`** – хендлеры бота (ввод/вывод).
- **`domain`** – бизнес-логика (управление пользователями, документы, RAG).
- **`infrastructure`** – внешние сервисы (Qdrant, DeepSeek API, Яндекс.Диск).
- **`core`** – общие утилиты, конфигурация, исключения.

### Ключевые технологии
- **Telegram Bot API** (Aiogram 3.11.0)
- **Qdrant** – векторная БД для эмбеддингов
- **DeepSeek API** – LLM для генерации ответов
- **Supabase** – таблица пользователей (роли: админ, руководитель, сотрудник)
- **Sentence Transformers** – модель эмбеддингов (`paraphrase-multilingual-MiniLM-L12-v2`)
- **FastAPI** (опционально) – веб-сервис для RAG

### Рабочий процесс
1. Сотрудник загружает документ (PDF/Word) в бота.
2. Бот сохраняет файл в хранилище (Яндекс.Диск или локально).
3. Документ парсится, разбивается на чанки, эмбеддинги сохраняются в Qdrant.
4. Сотрудник задаёт вопрос → RAG ищет релевантные чанки → DeepSeek генерирует ответ.
